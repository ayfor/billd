# billd app — UI kit

A high-fidelity, interactive recreation of the **billd** expense tracker
(dark-first, single user, CAD). Open `index.html`.

## Screens / surfaces
- **Sidebar.jsx** — left nav: logo lockup, Dashboard / Expenses / Budgets /
  Projections, Settings, account footer.
- **Topbar.jsx** — page title (pixel face), month select, `Add expense` with
  the `⌘K` hint.
- **DashboardView.jsx** — June overview: spent-vs-projection hero numerals,
  per-category budget bars, the spending-over-time chart, recent expenses.
- **SpendChart.jsx** — flat cumulative-spend chart: Sapphire actual series,
  Amethyst dashed projection, faint Lavender gridlines, today marker.
- **ExpensesView.jsx** — filterable table (search + category), right-aligned
  tabular amounts, month total.
- **QuickAddModal.jsx** — keyboard-first entry (amount, description, category,
  date defaulted to today; Enter saves). Opens on `⌘K`.
- **app.jsx** — wires view switching, the `⌘K` shortcut, and a save toast.
- **data.js** — the verbatim June 2026 sample data (amounts in integer cents).

## How it composes
Every surface is built from the design-system primitives on
`window.BilldDesignSystem_9669dd` (Card, Money, ProgressBar, Badge,
CategoryPill, NavItem, Button, Input, Select, Dialog, Table, PixelIcon,
PixelDivider). The kit adds layout and state only — it never re-implements a
primitive. `PixelIcon` renders inline SVG, so no icon paths need wiring.

## Notes
- Money is stored as integer cents and rendered two-decimal, tabular,
  right-aligned.
- Over-budget (Dining out, and the projection) reads in calm **Amethyst** —
  never red.
- One ornamental `PixelDivider` per view (on the Dashboard).
- `index.html` loads `../../_ds_bundle.js` (generated by the compiler) before
  the Babel screen scripts.
