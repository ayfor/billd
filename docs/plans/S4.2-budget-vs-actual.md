# S4.2 — Budget vs actual display

> Story: [Notion S4.2](https://app.notion.com/p/37ca7227e26f811ba62ee5e4b1b590ec) · Design: [F4 Budgets](https://app.notion.com/p/37ca7227e26f81a9a97bdcb7ee468f6e) · Design system: [canonical](https://app.notion.com/p/37da7227e26f81f28e85fa5c6d1d38f8) · PR: #11 (base `main`)

## Objective

Compute budget-vs-actual and render it: pixelated segment bars (Seaweed under / Amethyst over, capped at 100%), status lines, urgency sort. The computation is one exported function that S5.1's dashboard tiles reuse — single source of truth. Completes feature F4.

## Data Model

No schema change.

## Approach

1. **`src/lib/queries/budgetStatus.ts`** — the shared computation (this is the source of truth):
   - `budgetStatuses(userId, now)`: for each budget, compute `spentCents` (sum of that category's expenses in the current calendar month for `monthly`, current calendar year for `yearly`; UTC-aligned, consistent with `expenseQuery`), then derive `{ budget, category, spentCents, budgetCents, ratio, over, remainingCents, status }`. Pure given the queried rows; the date math is unit-tested via a helper `computeStatus(spentCents, budgetCents)`.
   - Sort by `ratio` descending (hottest first).
2. **Budgets page** — replace the plain rows with `<BudgetCard>`s rendering: category pill + timespan chip, "$spent / $budget", a **pixel segment progress bar** (fill ratio capped at 100%, Seaweed under / Amethyst at-or-over), and a status line ("Under by $X" Seaweed / "Over by $X" Amethyst). Keep the existing create/edit modal (S4.1) — cards open it on click.
3. **`src/components/BudgetBar.tsx`** — the pixelated segment bar (reuses the dashboard token style; Seaweed/Amethyst by `over`).
4. Header summary: Σ spent across budgeted categories / Σ budget amounts (current month).

## Computation contract (Design F4)

| Value | Formula |
|---|---|
| spent (monthly) | Σ expenses in category, current calendar month (UTC) |
| spent (yearly) | Σ expenses in category, current calendar year (UTC) |
| ratio | spent / budget (for the bar; bar caps at 1.0) |
| over | spent > budget |
| status | over → "Over by $(spent−budget)" (Amethyst); else "Under by $(budget−spent)" (Seaweed) |
| sort | ratio desc |
