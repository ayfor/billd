# S3.2 — Category usage stats & delete guard

> Story: [Notion S3.2](https://app.notion.com/p/37ca7227e26f81f1bad6eb0e63c07bf7) · Design: [F3 Categories](https://app.notion.com/p/37ca7227e26f81b99ee7e07c34635427) · Design system: [canonical](https://app.notion.com/p/37da7227e26f81f28e85fa5c6d1d38f8) · PR: #9 (base `main`)

## Objective

Add per-category usage stats (all-time count + current-month total) to the cards, and a strict delete guard: a category referenced by any expense (or, later, budget / recurring template) can't be deleted. Completes feature F3.

## Data Model

No schema change. `Expense.categoryId` is already `onDelete: Restrict` (S2.1), so the DB refuses the delete; this story adds the friendly pre-check + UI.

## Approach

1. **`src/lib/queries/categoryUsage.ts`** — `categoryUsage(userId)`: for each category, `_count` of expenses (all time) + sum of `amountCents` for the current calendar month (America/Toronto via UTC month bounds, consistent with `expenseQuery`). Returns a map keyed by categoryId. (Budgets/recurring counts added when those models land — guard is written to include them then.)
2. **`src/app/(dashboard)/categories/actions.ts`** — add `deleteCategory(id)`: `requireUser`; count references (expenses now; budgets/recurring when present); if > 0 → return `{ blocked, reason }` with counts; else `deleteMany where id+userId`; revalidate. Catch the Prisma `Restrict` violation (P2003) as a backstop → blocked.
3. **Categories page** — pass usage to `CategoriesClient`; cards show "{n} expenses · {formatCents(monthTotal)} this month".
4. **`CategoriesClient`** — add a **Delete** link with inline confirm (no browser dialogs); on blocked, show the explanation ("Used by N expenses — reassign or remove those first") inline; the seeded defaults are ordinary categories (deletable when unused).
