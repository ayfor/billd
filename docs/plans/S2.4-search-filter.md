# S2.4 — Search, filter & sort expenses

> Story: [Notion S2.4](https://app.notion.com/p/37ca7227e26f814c81e4f2bfc1686c45) · Design: [F2 Expenses](https://app.notion.com/p/37ca7227e26f81a29b6bd8688d11cc1c) · Design system: [canonical](https://app.notion.com/p/37da7227e26f81f28e85fa5c6d1d38f8) · PR: #7 (base `main`)

## Objective

Make the Expenses list searchable, filterable, and sortable — all URL-param-driven (shareable, refresh- and back-button-safe) — completing feature F2.

## Data Model

No schema change. Adds query-building from search params.

## Approach

1. **`src/lib/expenseQuery.ts`** — pure helpers: `parseExpenseParams(searchParams)` → `{ q, categoryId, month, sort, dir, page }` with safe defaults (sort=date, dir=desc, page=1); `buildWhere(userId, params)` → Prisma where (description `contains` case-insensitive; `categoryId` equals; `month` → date range `[start, nextMonth)`); `buildOrderBy(params)` → Prisma orderBy (date|amount, asc|desc, tie-break createdAt desc). Unit-tested.
2. **`src/components/ExpenseFilters.tsx`** (client) — the filter row from the F2 frame: search input (debounced 300ms → `?q=`), Category `<select>` ("All" default), Month `<select>` (recent months + "All time"), Sort `<select>` (Date ↓/↑, Amount ↓/↑). Each change does `router.push` with merged params and **resets `page` to 1**. Uses `useSearchParams`/`usePathname`/`useRouter`.
3. **Expenses page** — read params via `parseExpenseParams`, `count`/`findMany`/`aggregate` with the built where+orderBy, render `<ExpenseFilters>` above the list. Summary line + total reflect the filtered set.
4. **Empty states** — distinguish: zero expenses *ever* → first-run panel (existing); zero *matching* the filters → "No expenses match" panel with a **Clear filters** action (links to `/expenses`).

## Behaviour (Design F2 spec table)

| Behaviour | Spec |
|---|---|
| Search | case-insensitive substring on description; 300ms debounce; server-side `contains` |
| Default sort | date desc, tie-break createdAt desc |
| Sortable | date ↑↓, amount ↑↓ |
| Filters | category (single, "All" default), month (calendar month, "All time" option) |
| Pagination | 25/page (S2.1); resets to page 1 on any search/filter/sort change |
| State | all in URL search params — shareable, refresh-safe, back/forward-safe |
