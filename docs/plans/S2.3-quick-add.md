# S2.3 — Quick-add (⌘K)

> Story: [Notion S2.3](https://app.notion.com/p/37ca7227e26f817cb8fcd0998c877c52) · Design: [F2 Expenses](https://app.notion.com/p/37ca7227e26f81a29b6bd8688d11cc1c) · Design system: [canonical](https://app.notion.com/p/37da7227e26f81f28e85fa5c6d1d38f8) · PR: #6 (base `main`)

## Objective

Make expense entry global and fast: ⌘/Ctrl-K opens the quick-add modal from any authenticated page, with the category defaulting to most-recently-used, a live amount mask, and Esc to close.

## Scope note

The save-and-clear rapid-entry loop (AC-3) was delivered early by the **"Create another" checkbox** in S2.2. This story covers the remaining quick-add ACs: the global ⌘K shortcut (AC-1), most-recently-used category default (AC-2), Esc-closes + page reflects new expenses (AC-4), and the amount input mask (AC-5). Create-another remains the rapid-entry mechanism.

## Data Model

No schema change. Adds a "most-recently-used category" query (the category of the user's latest expense).

## Approach

1. **`src/components/QuickAdd.tsx`** (client) — a provider mounted in the dashboard layout that:
   - registers a global `keydown` listener: `(meta|ctrl)+K` → open create modal (preventDefault to override the browser); `Escape` → close;
   - renders the shared `ExpenseModal` in `create` mode when open;
   - exposes `openQuickAdd()` via context so in-page buttons (e.g. Expenses "Add expense") trigger the same single modal.
2. **`src/app/(dashboard)/layout.tsx`** — fetch the user's categories + most-recently-used category id (latest expense's category), pass both to `<QuickAdd>` wrapping `{children}`.
3. **`src/lib/queries/recentCategory.ts`** — `mostRecentCategoryId(userId)`: the category of the latest expense, else the first category by sortOrder. Unit-testable selector.
4. **`ExpenseModal`** — add: optional `defaultCategoryId` (used by quick-add); `Escape` keydown closes; **amount mask** — on input, strip non-digit/dot, enforce ≤2 decimals, render thousands separators (parse already strips commas, so submit is unaffected).
5. **`ExpensesClient`** — route its "Add expense" button through `openQuickAdd()` (one global create modal); keep the row-click **edit** modal local (edit isn't global).

## Sequence — ⌘K quick add

```mermaid
sequenceDiagram
    actor User
    participant W as window (keydown)
    participant Q as QuickAdd provider
    participant M as ExpenseModal (create)
    participant A as createExpense action
    User->>W: ⌘K (any dashboard page)
    W->>Q: open (preventDefault)
    Q->>M: render, focus Amount, category = most-recent
    User->>M: type amount (masked) + details, Enter
    M->>A: createExpense
    A-->>M: ok → revalidate; close or (Create another) reset+stay
```
