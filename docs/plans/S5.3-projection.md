# S5.3 — Spend projection

> Story: [Notion S5.3](https://app.notion.com/p/37ca7227e26f812fb76ef0192a6b96d2) · Design: [F5 Dashboard](https://app.notion.com/p/37ca7227e26f81b6bc13d40581dec33e) · Design system: [canonical](https://app.notion.com/p/37da7227e26f81f28e85fa5c6d1d38f8) · PR: #14 (base `main`)

## Objective

Add the projection panel — billd's core premise: linear-pace projected month-end spend (Pixelify numeral), the "too early to call" guard under 3 days elapsed, and an on-pace / trending-over status. Reuses the `projectMonthEnd` helper from S5.2 (single source of truth — the chart's projection endpoint and this numeral can never disagree). Completes feature F5.

## Data Model

No schema change.

## Approach

1. **`src/lib/queries/spendSeries.ts`** — add `projectionStatus(spentCents, expectedCents, now)`: returns `{ tooEarly, projectedCents, onPace, label, tone }`:
   - `tooEarly` = `now.getUTCDate() < 3` → panel shows "Too early to call" (Lavender 50%), no number;
   - else `projectedCents = projectMonthEnd(...)` (the existing helper); `onPace = projectedCents <= expectedCents`;
   - label/tone: on pace → "On pace · under budget" (Seaweed); over → "Trending over · review budgets" (Amethyst).
   - Pure → unit-tested incl. month boundaries / leap Feb / DST safety (UTC math).
2. **Dashboard page** — render a Projection panel beside the hero (hero row becomes a 2-up): Pixelify projected numeral (or "Too early to call"), sub-label, status line. When no budgets, the existing hero CTA already covers it; projection still shows the projected spend with a neutral status.
3. **`src/components/ProjectionPanel.tsx`** — presentational; pixel numeral + status.

## Computation contract (Design F5)

| Value | Formula |
|---|---|
| Projected | `projectMonthEnd` (spent ÷ daysElapsed × daysInMonth) — shared with S5.2 chart |
| Too early | daysElapsed < 3 → "Too early to call", no numeral |
| Status | projected ≤ expected → Seaweed "On pace · under budget"; else Amethyst "Trending over · review budgets" |
