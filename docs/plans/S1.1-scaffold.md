# S1.1 — Scaffold, CI & deployment baseline

> Story: [Notion S1.1](https://app.notion.com/p/37ca7227e26f819ba59bdd3a0789e210) · Design: [F1 Accounts](https://app.notion.com/p/37ca7227e26f81c78790f25db76505e7) · Design system: [canonical](https://app.notion.com/p/37da7227e26f81f28e85fa5c6d1d38f8) · PR: #1

## Objective

Stand up `ayfor/billd` as a working skeleton — Next.js (App Router, TS), Tailwind 4, shadcn/ui, Prisma, the billd design tokens, test tooling (Vitest + RTL + Playwright), GitHub Actions CI, and the pipeline's `docs/` scaffold — so every subsequent story starts from a bootable, tested, deployable base.

## Endpoints

None — no API surface in this story.

## Data Model

None — no models yet. `docs/architecture/erd.md` is created as a stub; first entities land in S1.2 (User) and S2.1 (Expense, Category).

## Approach

1. `create-next-app` (TS, App Router, `src/` dir, Tailwind, ESLint) on Node 22.
2. Design tokens: copy `tokens/*.css` + `assets/icons` + `logo-mark.svg` + `pixel-divider.svg` from the `design-assets` branch's `design-system/` into `src/app/tokens/` and `public/icons/`; import in `globals.css`. Fonts (Silkscreen, Pixelify Sans, Inter) via `next/font/google`.
3. shadcn/ui init (default style, CSS variables) — components added per-story, not in bulk.
4. Prisma init: empty schema + generated client wiring in `src/lib/prisma.ts`; `.env.example` documents `DATABASE_URL` (Supabase). No live DB connection required by this story's ACs.
5. Test tooling: Vitest + RTL (jsdom) with one real smoke test (landing renders the wordmark on the Shadow Grey canvas); Playwright config checked in with one e2e smoke spec, wired as a separate non-blocking CI job (browser install is slow; it becomes blocking when S1.2 ships real flows).
6. CI: `.github/workflows/ci.yml` — `lint`, `test`, `build` on every PR (blocking); `e2e` job allowed-to-fail until S1.2.
7. Landing page: minimal — canvas, Silkscreen wordmark, pixel divider, palette proof (the AC-1 visual check).
8. Vercel: standard zero-config Next app. **Deploy requires Josh to link the repo in the Vercel dashboard (no CLI auth on this machine)** — flagged as the one externally-blocked AC.

## Deviations

- **AC-3 (Vercel deploy) descoped** — Josh's call 2026-06-13: external deployment is out of scope for the pilot. The app stays deploy-ready (zero-config Next on Vercel) but no deploy is required to satisfy this story. Re-add later if/when billd goes live.
- **Prisma 7 adapter:** Prisma 7 removed `url` from the schema datasource and requires a driver adapter at the client. Added `@prisma/adapter-pg`; `src/lib/prisma.ts` constructs `PrismaClient` with `PrismaPg`. Connection URL lives in `prisma.config.ts` via `DATABASE_URL`. Generated client is gitignored (regenerated on `postinstall`).
- **Next.js 16 / React 19:** `create-next-app` pulled Next 16.2.9 + React 19.2.4 (newer than the "14+" the design notes assumed). No story impact; App Router APIs unchanged.
- Vitest `globals: true` enabled so RTL auto-cleanup runs between tests.
